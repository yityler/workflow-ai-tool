5. Run it through the following: `python -m prototype.main`

## Module Summary
### 1. The Interface & Context Layer (`prototype/input/`)
This module handles how the user interacts with the system and how initial ideas are refined before generation
* **`input_ui.py` & `input_main.py`:** Provides the interface layer through which a user defines product requirements, goals, or design aesthetics.
* **`input_rag.py`:** Implements Retrieval-Augmented Generation. It pulls contextual information or design inspirations from a local knowledge base to inject relevant background data into the generation loop.
* **`input_build_prompt.py` & `input_build_prompt_image.py`:** Acts as the prompt-engineering layer. It translates raw user goals and RAG context into structured, optimized prompts designed to yield deterministic results from text and image AI models.

### 2. The API & Execution Backend (`prototype/product_generation/`)
Powered by **FastAPI** and **Uvicorn**, this is the execution powerhouse that processes layout architectures and hooks into external generative APIs.
* **`cornerstone_backend.py`:** Serves as the central API gateway, managing requests for layout creation, image rendering, and block manipulation.
* **`cornerstone_page_generator.py` & `cornerstone_blocks.py`:** Works with `google-genai` to dynamically synthesize UI wireframes or document structures using predefined structural components ("blocks").
* **`image_gen.py` & `image_gen_hf.py`:** Handles multi-provider image asset creation, leveraging Hugging Face endpoints alongside Google's stack to generate customized visual elements.
* **`cornerstone_renderer.py` & `preview_server.py`:** Compiles the structural data models generated by the LLM into visual mockups and spins up a local server for real-time validation and review.

### 3. Page Assembly & Optimization Workspace (`prototype/Tyler_Yi_Prototype_Folder/`)
This workspace functions as the fine-tuning and compilation engine to ensure code output is clean, cost-efficient, and adheres to strict design guardrails.
* **`page_generator.py`:** Aggregates the generated structural blocks, styling variables, and generated image assets into a cohesive final layout product.
* **`token_optimizer.py`:** Optimizes prompt lengths and structural code inputs/outputs to prevent context-window bloat, lowering API costs and latency.
* **`design_knowledge.txt`:** Acts as an immutable ruleset of design principles (e.g., color theories, spacing conventions, layout rules) used to guide the generation models.
